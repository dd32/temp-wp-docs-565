https://developer.wordpress.org/themes/theme-security/using-nonces/

<p>WordPress nonces are one-time use security tokens generated by WordPress to help protect URLs and forms from misuse.</p>
<p>If your theme allows users to submit data; be it in the Admin or the front-end; nonces can be used to <strong>verify a user intends to perform an action</strong>, and is instrumental in protecting against <a href="https://developer.wordpress.org/themes/theme-security/common-vulnerabilities/#cross-site-request-forgery-csrf">Cross-Site Request Forgery(CSRF)</a>.</p>
<p>An example is a WordPress site in which authorized users are allowed to upload videos. As an authorized user uploading videos is an intentional action and permitted. However, in a CSRF, a hacker can hijack (forge) the use of an authorized user and perform a fraudulent submission.</p>
<p>The one-time use hash generated by a nonce, prevents this type of forged attacks from being successful by validating the upload request is done by the current logged in user. Nonces are unique only to the current user's session, so if an attempt is made to log in or out any nonces on the page become invalid.</p>
<h2>Creating a Nonce</h2>
<ul>
<li>wp_nonce_url() - Adding a nonce to an URL.</li>
<li>wp_nonce_field() - Adding a nonce to a form.</li>
<li>wp_create_nonce() - Using a nonce in a custom way; useful for processing AJAX requests.</li>
</ul>
<h2>Verifying a Nonce</h2>
<ul>
<li>check_admin_referer() - To verify a nonce that was passed in a URL or a form in an admin screen.</li>
<li>check_ajax_referer() - Checks the nonce (but not the referrer), and if the check fails then by default it terminates script execution.</li>
<li>wp_verify_nonce() - To verify a nonce passed in some other context.</li>
</ul>
<h2>Example</h2>
<p>In this example, we have a basic submission form.</p>
<p><strong>Create the Nonce</strong></p>
<p>To secure your form with a nonce, create a hidden nonce field using wp_nonce_field() function:</p>
<p>[php] <br />&lt;form method="post"&gt;</p>
<p>    &lt;!-- some inputs here ... --&gt;</p>
<p>    &lt;?php wp_nonce_field( 'name_of_my_action', 'name_of_nonce_field' ); ?&gt;</p>
<p>&lt;/form&gt;</p>
<p>[/php]</p>
<p><strong>Verify the Nonce</strong></p>
<p>In our example we first check if the nonce field is set since we do not want to run anything if the form has not been submitted. If the form has been submitted we use the nonce field value function. If nonce verification is successful the form will process.</p>
<p>Using wp_verify_nonce + wp_nonce_ays <em>(Are You Sure)</em></p>
<p>[php]</p>
<p>if (</p>
<p>    ! isset( $_POST['name_of_nonce_field'] )</p>
<p>    || ! wp_verify_nonce( $_POST['name_of_nonce_field'], 'name_of_my_action' )</p>
<p>) {</p>
<p>    wp_nonce_ays( '' );</p>
<p>}</p>
<p>// process form data</p>
<p>[/php]</p>
<p>or using check_admin_referer</p>
<p>[php]</p>
<p>check_admin_referer( 'name_of_my_action', 'name_of_nonce_field' );</p>
<p>// process form data</p>
<p>[/php]</p>
<p>In these examples the basic nonce process:</p>
<ol>
<li>Generates a nonce with the wp_nonce_field() function.</li>
<li>The nonce is submitted with the form submission.</li>
<li>The nonce is verified for validity using the wp_verify_nonce() or check_admin_referer() function. If not verified the request exits with a default error message (don't precise the error message).</li>
</ol>
